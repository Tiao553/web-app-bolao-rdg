09 — Notas

Pontos de Atenção

Server vs Client Components

Todas as páginas são Server Components. ScoreGauge, TagCloud, MatchCard (com inputs) e TagCloud precisam de 'use client'. Criar como Client Components filhos e importar nas páginas — nunca adicionar 'use client' no arquivo de página diretamente.

PostCSS obrigatório

Mantine requer postcss-preset-mantine em postcss.config.cjs na raiz do app/. Sem isso as variáveis --mantine-color-* não são geradas e os componentes ficam sem cor no build de produção.

Lógica intacta

Nenhum arquivo de negócio é alterado. access-gate.tsx, session.ts, api-client.ts e todos os guards do backend permanecem inalterados. O redesign é 100% aditivo — só substitui HTML puro por componentes Mantine.

Mantine + Next.js 15 App Router

Usar ColorSchemeScript no <head> do layout.tsx para evitar flash de tema na hidratação. Mantine v7 tem suporte nativo a App Router. Não é necessário next/dynamic — apenas separar Client Components corretamente.

Background mais escuro que o Mantine default

O Archlify usa #09090B — mais escuro que o dark[9] padrão do Mantine (#141517). Forçar no CSS global: body { background-color: #09090B } e sobrescrever --mantine-color-body no :root.

ScoreGauge — dados do backend

ScoreGauge e DimensionBars recebem score, max, rank e breakdown como props — dados já calculados pelo backend. O componente é puramente visual, sem lógica de cálculo. Endpoint: GET /api/member/ranking/me.
